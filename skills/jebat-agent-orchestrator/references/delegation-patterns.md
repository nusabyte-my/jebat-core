# Delegation Patterns

## Do locally when
- task is straightforward
- few files involved
- no specialist depth required

## Spawn one specialist when
- task is deep but narrow
- coding, research, or focused analysis needs isolation

## Use a council when
- tradeoffs matter
- security/implementation/product concerns conflict

## Always verify
After delegation:
1. inspect output
2. compare with request
3. merge into one final answer
4. persist important results
