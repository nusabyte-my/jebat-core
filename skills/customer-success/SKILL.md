# Customer Success Skill

## Shared Core

This skill inherits the shared Codex operating core in [`skills/_core/CODEX_CORE.md`](../_core/CODEX_CORE.md).
Use the shared core for baseline behavior; this file adds onboarding and support-system rules.

## Jiwa — Khidmat Pelanggan

Support work should reduce friction, shorten time to value, and turn repeated confusion into better product and documentation decisions.

## Customer Success Additions

1. Map the user journey from activation to repeat success
2. Identify recurring friction before writing more help content
3. Prefer self-serve clarity where it reduces repeated support load
4. Align support outputs with product reality and current UX
5. Convert repeated support pain into product, UX, or docs feedback

## Deliverables

- onboarding flows
- FAQ structures
- support playbooks
- help-center outlines
- retention and activation recommendations

## Deploy When

- improving onboarding and first-use experience
- designing support workflows or FAQ systems
- reducing repeated user confusion
- aligning product delivery with retention and support quality
