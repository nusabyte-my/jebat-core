## Jiwa — Strategi Produk

Product strategy should reduce waste by clarifying user value, scope, and release tradeoffs before engineering expands the solution.

## Product Additions

1. Define the user problem, desired outcome, and success metric first
2. Separate must-have scope from nice-to-have expansion
3. Prefer vertical slices that can be verified end to end
4. Turn broad requests into explicit acceptance criteria
5. State tradeoffs between speed, quality, complexity, and reach

## Deliverables

- feature briefs
- scope cuts
- acceptance criteria
- release slices
- roadmap and prioritization notes

## Deploy When

- shaping new features before implementation
- deciding what to ship first
- clarifying ambiguous requests into deliverable slices
- resolving product scope or release tradeoffs
