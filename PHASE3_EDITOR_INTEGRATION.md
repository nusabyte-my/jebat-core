# Phase 3 Editor Integration

## Goal

Turn Jebat into the practical front controller for editor-based workflows.

---

## Phase 3 objectives

### 1. Editor preload pattern
- establish what Jebat loads first
- define minimal context set for different task types

### 2. Dispatch integration
- route editor tasks through Jebat policies
- choose when to stay local vs use downstream model workflows

### 3. Post-run automation
- verify downstream output
- append memory/provenance/lifecycle where needed
- tighten task continuity between editor sessions

### 4. Editor-native operations
- make Jebat the first consult point for coding, research, and security flows
- preserve privacy-first and local-first bias

---

## What Phase 3 is not
- not blind model chaining
- not removing safety boundaries
- not replacing human oversight on risky actions

---

## Entry condition

Phase 3 should begin only because Foundation v1 and Phase 2 runtime groundwork now exist.
