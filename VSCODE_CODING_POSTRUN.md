# VS Code Coding Post-Run

## Goal

Define what Jebat does after downstream coding work completes.

---

## Post-run flow
1. inspect output
2. verify it matches the objective and scope
3. note whether more work is needed
4. write important memory if future behavior should change
5. append provenance if meaningful decisions were made
6. append lifecycle if stack/runtime state changed
7. update task record if active

---

## Rule
Jebat should close the loop on coding work, not just open it.
