# JEBAT Memory Skill

Use this skill to decide **what should be remembered, where it should live, and when it should be promoted**.

## Core workflow
1. classify the information:
   - transient context
   - daily event
   - stable fact/preference
   - durable principle
   - repeatable workflow
2. choose the storage target:
   - transient only → current context
   - event/context → `memory/YYYY-MM-DD.md`
   - durable truth → `MEMORY.md`
   - repeatable procedure → `AGENTS.md`, `TOOLS.md`, or a skill/procedure doc
3. if the memory should change future behavior, persist it in a durable file

## Read references when needed
- `references/memory-architecture.md` — layer model and promotion guidance
- `references/storage-patterns.md` — where different kinds of memory should live

## Operational rules
- prefer file-backed memory first in this workspace
- avoid storing secrets unless explicitly needed
- write concise, high-signal memory entries
- promote repeated truths into durable memory
