JEBAT Enterprise Core — Private Inference Router

Codename: Hang Jebat · v6.1.0-stable Production

High-performance sovereign AI cognitive router running behind corporate firewalls. Manages private inference routing to 7 LLM API gateways or fully offline models via Ollama. Implements persistent 5-layer recall context storage (Sensory → Episodic → Semantic → Conceptual → Procedural) with active vector heat-scoring, a secure local keyring, and 3-tier command classification guardrails. Built with FastAPI, Uvicorn, SQLite, and Redis caching.

✓ 7 Reasoning Modes (Ultra-Think) ✓ 5-Layer Heat-Scored Memory ✓ Secure Local Keyring ✓ Native MCP Client + Server ✓ 3-Tier RBAC Guardrails ✓ 170 Tests Passing ✓ Air-Gap Compatible ✓ Ollama / llama.cpp Offline ✓ OpenAI-Compatible Proxy

6-Step Operational Onboarding

From Zero to Sovereign Cluster

The complete enterprise onboarding guide. Follow all six steps to deploy a production-grade private AI workstation.

Step 01 / Zero-Config Installation

Install the Platform

Five install methods — from zero-config npx to air-gapped source. Choose what fits your environment.

# ✦ Option 0: Zero-config (recommended)
npx @nusabyte/jebat repl
bunx @nusabyte/jebat repl

# Option A: Global npm install
npm install -g @nusabyte/jebat

# Option B: pip install
pip install jebat

# Option C: Docker
docker compose up -d

# Option D: Source (air-gapped)
git clone github.com/nusabyte-my/jebat-core
Step 02 / Credential Hardening

Initialize Encrypted Keyring

Securely encrypt all local API credentials. JEBAT uses a local keyring to prevent key leakage — no plaintext secrets on disk.

jebat auth set openai <key>
jebat auth set anthropic <key>
jebat auth set ollama http://localhost:11434
Step 03 / Configure Cognitive Router

Set Provider & Fallback Chain

Configure the cognitive router in ~/.jebat/config.yaml. Set primary provider, fallback chain, and safety tier defaults.

llm:
  provider: ollama
  model: qwen2.5-coder:7b
  fallback_providers:
    - openai
    - anthropic
Step 04 / Run Agent Loops

Execute Autonomous Tasks

Launch the ReAct reasoning engine for one-shot queries or drop into the interactive REPL for conversational workflows.

jebat agent "Verify the build config in package.json"

# Interactive streaming REPL
jebat chat-repl
Step 05 / Orchestrate Swarms

Delegate to Specialist Agents

Deconstruct complex goals into parallel subtasks and dispatch them to role-based agents (Tukang, Hulubalang, Pawang).

jebat delegate run "Audit all API endpoints in src/services" \
  --tools terminal,file
Step 06 / Deploy Channels

Expose to Team Messaging

Connect your agent gateway to team platforms. Let operations query logs and dispatch commands from Telegram, Slack, or Discord.

jebat social start \
  --channels telegram,slack

Platform Architecture

Technical Infrastructure Overview

Engineered for absolute data residency, compliance, and auditability at every layer. Zero external telemetry.

01 / Cognitive Routing

Governed 7-Mode Reasoning

Direct LLM traffic based on complexity using 7 distinct Ultra-Think profiles: Fast, Deliberate, Deep, Strategic, Creative, Critical, and Custom. Prevents costly routing for basic ops.

02 / Eternal Memory

5-Layer Heat-Scored Recall

Preserve system logs, past audits, and operational context across 5 persistence tiers in SQLite with active vector heat-scoring. Response time <100ms per memory retrieval.

03 / Security Sandbox

3-Tier RBAC Guardrails

Classify tool execution into AUTO (read-only), CONFIRM (write/modify requiring validation), and DANGEROUS (destructive requiring explicit approval). Full audit trail.

04 / MCP Protocol Bus

Standard Integration Layer

Native MCP client + server running over stdio or HTTP transport. Exposes 13 core capabilities to any MCP-compatible IDE, CLI, or automation workflow. OpenAI-compatible proxy at :20128.

05 / Agent Swarm Engine

17 Specialized Agent Roles

Role-based multi-agent orchestration: Tukang (builders), Hulubalang (executors), Pawang (debuggers), Pengacara (reviewers), Bendahara (planners) + 12 domain specialists.

06 / Provider Failover

6-Provider Cognitive Routing

Automatic failover across OpenAI, Google, Anthropic, OpenRouter, Ollama, and llama.cpp. Health monitoring, cost tracking, and budget guardrails built in. OpenAI-compatible API proxy.

Safety Classification

3-Tier Execution Guardrails

JEBAT categorizes all tool execution into permission tiers to safeguard target systems. Zero destructive commands without explicit authorization.

AUTO

Read-Only Safe

Commands executed without intervention. Non-destructive, read-only operations that cannot modify system state.

cat grep ls status git log jebat health
CONFIRM

Write / Modify

Commands prompting user validation before execution. Write, modification, and network operations requiring explicit confirmation.

write patch git commit npm install jebat deploy
DANGEROUS

Destructive / Escalated

Commands requiring explicit terminal validation tags before proceeding. Destructive or privilege-escalating operations with full audit log entry.

rm -rf sudo DROP TABLE chmod 777 iptables
