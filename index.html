Platform Architecture

Governed Inference at Scale

Every component designed for auditability, data sovereignty, and operational control.

01 / Reasoning

Ultra-Think Engine

Seven cognitive modes — fast, deliberate, deep, strategic, creative, critical, direct. Real LLM calls with streaming output. Choose the depth your task demands.

02 / Persistence

Eternal Memory System

Context persists across sessions, projects, and products. Accumulates institutional knowledge instead of resetting. Layered storage with heat-scored retrieval.

03 / Providers

Provider-Agnostic LLM Layer

Ollama for fully offline operation, or plug in OpenRouter, Anthropic, OpenAI, Google Gemini. Automatic failover chain. Switch providers without changing workflow.

04 / Protocol

MCP Compliance

Model Context Protocol server for standardized tool integration. Skills adapter converts SKILL.md definitions into registered tools over stdio or HTTP.

05 / Agents

Domain Sub-Agents

17 purpose-built agents: Tukang (build), Hulubalang (security), Pawang (research), Syahbandar (ops), Bendahara (data), Penyemak (QA). Orchestrated by Core.

06 / Sovereignty

Zero Cloud Dependency

Default config runs on local hardware via Ollama. No API keys required. Graceful degradation. 57 tests pass with zero external services. Air-gap ready.

Why It Matters

Sovereign AI Explained

Why Organizations Are Moving to Sovereign AI

Cloud-based AI services require sending proprietary data, internal communications, and sensitive code to third-party servers. Every prompt becomes a data point in someone else's infrastructure. Pricing changes without notice. Models get deprecated. Access can be revoked. Sovereign AI eliminates these risks by running models on hardware you own, behind firewalls you control.

JEBAT was built for this reality. The default installation uses Ollama to run open-weight models locally — no API keys, no billing accounts, no outbound connections. When you need more capability, plug in any of six commercial providers as an upgrade, not a requirement. The platform treats cloud AI as an optional accelerator, not a prerequisite.

This architecture matters for regulated industries, government agencies, and any organization where data residency requirements make cloud AI non-viable. With JEBAT, compliance isn't a feature toggle — it's the default state. Your models stay on your network. Your memory stays in your database. Your reasoning pipeline never touches infrastructure you don't audit.
