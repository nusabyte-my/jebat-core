JEBAT Enterprise Core — Private Inference Router

Codename: Hang Jebat · v6.1.0-stable Production

High-performance sovereign AI cognitive router running behind corporate firewalls. Manages private inference routing to 6 LLM API gateways or fully offline models via Ollama. Implements persistent 5-layer recall context storage (Sensory → Episodic → Semantic → Conceptual → Procedural) with active vector heat-scoring, a secure local keyring, and 3-tier command classification guardrails. Built with FastAPI, Uvicorn, PostgreSQL, and Redis caching. All data remains on-premise with zero external telemetry.

✓ 7 Cognitive Reasoning Modes (Ultra-Think) ✓ 5-Layer Heat-Scored Memory ✓ Secure Local Keyring ✓ Native MCP Client + Server ✓ 3-Tier RBAC Guardrails ✓ 170+ Tests Passing ✓ Air-Gap Compatible ✓ Ollama / llama.cpp Offline ✓ OpenAI-Compatible Proxy

JEBAT Ghost Database — Ephemeral Data Isolation

Codename: Spectre · v1.0.0 Live

Enterprise-grade ephemeral database forking engine purpose-built for isolated agent workspaces. Create surgical database forks from production, execute parameterized SQL against sandboxed copies, snapshot state at arbitrary checkpoints, and auto-reclaim resources on completion — ensuring zero data contamination between agent execution contexts.

✓ Fork Production Databases ✓ Ephemeral Agent Workspaces ✓ Parameterized SQL Execution ✓ Arbitrary Checkpoints ✓ Automatic Resource Cleanup ✓ SQL History Audit Trail ✓ 14 Registered MCP Tools

Ghost Database Operations:

# 1. Check Ghost CLI availability $ jebat ghost status # 2. Create an ephemeral database fork $ jebat ghost create agent-workspace-1 # 3. Fork production data for safe experimentation $ jebat ghost fork production-db agent-sandbox # 4. Execute SQL against the isolated workspace $ jebat ghost sql agent-sandbox "SELECT * FROM users LIMIT 10" # 5. Create a checkpoint before destructive operations $ jebat ghost checkpoint agent-sandbox pre-refactor # 6. Restore from checkpoint on failure $ jebat ghost checkpoint restore agent-sandbox pre-refactor # 7. Reclaim resources from idle workspaces $ jebat ghost cleanup --hours 24

JEBAT Catalyst Observability — Distributed Tracing & HALO Analysis

Codename: Resonance · v1.0.0 Live

Production-grade distributed tracing and performance intelligence engine for multi-agent operations. Instrument any service with zero-config span collection, correlate cross-service traces in real-time, and apply HALO (High-Altitude Lens Optimization) analysis to surface bottleneck patterns, latency anomalies, and resource contention across your entire sovereign infrastructure.

✓ Distributed Tracing ✓ Zero-Config Instrumentation ✓ Span Lifecycle Management ✓ HALO Analysis Engine ✓ Cross-Service Correlation ✓ Latency Anomaly Detection ✓ 9 Registered MCP Tools

Catalyst Observability Operations:

# 1. Initialize Catalyst with Gateway $ jebat catalyst init --gateway # 2. Auto-instrument JEBAT services $ jebat catalyst instrument # 3. Start a named trace $ jebat catalyst trace start --name agent-search # 4. Add spans to track individual operations $ jebat catalyst span start --trace agent-search --name db-query $ jebat catalyst span end --trace agent-search --span db-query # 5. Run HALO analysis on completed traces $ jebat catalyst halo trace-1 trace-2 --type full # 6. List all collected traces $ jebat catalyst trace list

6-Step Operational Onboarding

From Zero to Sovereign Cluster

The complete enterprise onboarding guide. Follow all six steps to deploy a production-grade private AI workstation.

Step 01 / Zero-Config Install

Install the Platform

Run via npx with zero setup — the Node wrapper auto-installs the Python core on first run. Or install directly with pip.

# ✦ Zero-config (recommended)
npx @nusabyte/jebat repl
bunx @nusabyte/jebat repl

# Or install globally
npm install -g @nusabyte/jebat

# Or Python direct
pip install jebat
Step 02 / Configure

Set Provider & Preferences

Configure your LLM provider, model, and safety mode in ~/.jebat/jebat.yaml.

# Show current config
npx @nusabyte/jebat config show

# Set provider
npx @nusabyte/jebat config set agent.default_provider ollama
npx @nusabyte/jebat config set agent.default_model qwen2.5-coder:7b

# Or edit interactively
npx @nusabyte/jebat config edit
Step 03 / Launch REPL

Start Chatting

Launch the interactive REPL for streaming conversations with tool calling, memory, and multi-agent swarms.

# Interactive REPL
npx @nusabyte/jebat repl

# One-shot chat
npx @nusabyte/jebat chat "Explain the memory system"

# Run an agent task
npx @nusabyte/jebat agent "Audit src/services"
Step 04 / Use Tools

Explore Capabilities

JEBAT ships with 89 registered tools — file ops, git, browser, memory, web search, and more.

# List all tools
npx @nusabyte/jebat tools list

# Store a memory
npx @nusabyte/jebat memory store "Key insight"

# Search memories
npx @nusabyte/jebat memory search "insight"

# Check system status
npx @nusabyte/jebat status
Step 05 / MCP Integration

Connect Your IDE

Connect JEBAT as an MCP server to Cursor, VS Code, Zed, or Windsurf for in-editor AI assistance.

# Connect to an MCP server
npx @nusabyte/jebat mcp connect --stdio "python -m my_server"

# List connected servers
npx @nusabyte/jebat mcp list

# Call a tool
npx @nusabyte/jebat mcp call server tool \
  '{"param": "value"}'
Step 06 / Go Sovereign

Deploy Production

For production, deploy the Python core on a private VPS with Ollama or llama.cpp for fully air-gapped operation.

# Clone source for self-hosted deploy
git clone https://github.com/nusabyte-my/jebat-core.git
cd jebat-core
pip install -e .

# Docker deployment (coming Q3)
docker compose up -d

# Or sync to VPS
bash scripts/sync-to-vps.sh

Platform Architecture

Technical Infrastructure Overview

Engineered for absolute data residency, compliance, and auditability at every layer. Every byte of data stays on your infrastructure — zero external telemetry, zero cloud dependency. SOC2 and HIPAA audit-ready out of the box.

01 / Cognitive Routing

Governed 7-Mode Reasoning

Direct LLM traffic based on complexity using 7 distinct Ultra-Think profiles: Fast, Deliberate, Deep, Strategic, Creative, Critical, and Custom. Prevents costly routing for basic ops.

02 / Eternal Memory

5-Layer Heat-Scored Recall

Preserve system logs, past audits, and operational context across 5 persistence tiers in SQLite with active vector heat-scoring. Response time <100ms per memory retrieval.

03 / Security Sandbox

3-Tier RBAC Guardrails

Classify tool execution into AUTO (read-only), CONFIRM (write/modify requiring validation), and DANGEROUS (destructive requiring explicit approval). Full audit trail.

04 / MCP Protocol Bus

Standard Integration Layer

Native MCP client + server running over stdio or HTTP transport. Exposes 13 core capabilities to any MCP-compatible IDE, CLI, or automation workflow. OpenAI-compatible proxy at :20128.

05 / Agent Swarm Engine

17 Specialized Agent Roles

Role-based multi-agent orchestration: Tukang (builders), Hulubalang (executors), Pawang (debuggers), Pengacara (reviewers), Bendahara (planners) + 12 domain specialists.

06 / Provider Failover

6-Provider Cognitive Routing

Automatic failover across OpenAI, Anthropic, Ollama, llama.cpp, OpenRouter, and a local echo fallback. Health monitoring, cost tracking, and budget guardrails built in. Each provider registers as a first-class client with streaming support. OpenAI-compatible API proxy at any port.

Safety Classification

3-Tier Execution Guardrails

JEBAT categorizes all tool execution into permission tiers to safeguard target systems. Zero destructive commands without explicit authorization.

AUTO

Read-Only Safe

Commands executed without intervention. Non-destructive, read-only operations that cannot modify system state.

cat grep ls status git log jebat health
CONFIRM

Write / Modify

Commands prompting user validation before execution. Write, modification, and network operations requiring explicit confirmation.

write patch git commit npm install jebat deploy
DANGEROUS

Destructive / Escalated

Commands requiring explicit terminal validation tags before proceeding. Destructive or privilege-escalating operations with full audit log entry.

rm -rf sudo DROP TABLE chmod 777 iptables
