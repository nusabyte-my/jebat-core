# JEBAT Operations

## Goal

Give Jebat practical runtime routines that can be used immediately inside this workspace.

---

## 1. End-of-session routine

When a meaningful work block ends:
1. summarize what happened using `scripts/session-wrap-up-template.md`
2. update `memory/YYYY-MM-DD.md`
3. promote durable facts to `MEMORY.md`
4. update procedures if a new workflow was discovered

---

## 2. Memory maintenance routine

Use `scripts/memory-maintenance-checklist.md`.

Run when:
- daily notes accumulate
- repeated project decisions appear
- memory feels noisy
- heartbeat maintenance is appropriate

---

## 3. Orchestration routine

Use `scripts/orchestration-checklist.md` before spawning specialists.

Rule:
- local-first
- specialist second
- council only for meaningful tradeoffs

---

## 4. Security review routine

Use `scripts/security-review-checklist.md` when asked for:
- audits
- exposure reviews
- hardening priorities
- risk summaries

---

## 5. Memory write rule

Write things down when they affect:
- future decisions
- user preferences
- architecture choices
- recurring workflows
- unresolved follow-up tasks

---

## 6. Safe import rule

Before adapting external skills:
- use `EXTERNAL_SKILL_VETTING.md`
- use `TRUST_VERIFICATION.md`
- use `SKILL_VETTING_WORKFLOW.md`

---

## 7. Hermes runtime rule

For every request:
1. classify
2. recall if prior work matters
3. choose smallest correct tool/action
4. verify result
5. persist what matters
