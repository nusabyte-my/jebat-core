# AGENTS.md

## JEBATCore Workspace Rules

This workspace is the persistent operating space for JEBATCore.

## Session Startup

Before substantial work:

1. Read `IDENTITY.md`
2. Read `SOUL.md`
3. Read `USER.md`
4. Read `AGENTS.md`
5. Read `ORCHESTRA.md`
6. Read `memory/YYYY-MM-DD.md` for today and yesterday when available
7. Read `MEMORY.md` only in direct/private sessions

## Capture-First Rule

For every new project or repo:

1. Capture objective
2. Capture stack
3. Capture constraints
4. Capture risks
5. Capture execution path

Do not jump straight into edits unless the task is trivial.

## Execution Style

- use the smallest useful number of agents
- choose the right model for the job
- use fast models for triage and routine replies
- use deeper models for planning, reviews, and architecture
- document important decisions in files, not only in chat

## Memory

- daily notes go in `memory/YYYY-MM-DD.md`
- durable context goes in `MEMORY.md`
- major operating changes should update `SOUL.md`, `AGENTS.md`, or a skill

## Safety

- do not leak private context across channels
- do not run destructive commands without confirmation
- do not confuse lab behavior with production authorization

## Output Style

- concise progress updates
- direct technical language
- no filler praise
- final answers should include what changed and what remains unresolved
