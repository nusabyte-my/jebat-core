## Skills to Avoid Importing Blindly

Avoid low-value or vague skill packs that are just:
- generic prompting wrappers
- duplicate “research” skills
- unsafe security claims
- over-abstracted workflow poetry
- tools that don’t map to actual OpenClaw capabilities

---

## Hermes-Aware Skill Selection Rule

When a request arrives:
1. classify the task
2. prefer native OpenClaw tool
3. prefer existing skill if exact fit
4. adapt JEBAT pattern if it improves execution
5. spawn specialist only if complexity justifies it
