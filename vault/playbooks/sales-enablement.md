# Sales Enablement Playbook

Use this playbook for one-pagers, capability decks, objection handling, outbound support messaging, and reusable sales collateral.

## Trigger

Use when the goal is to support qualification, outbound messaging, sales clarity, or internal/client-facing commercial enablement.

## Primary Roles

- Panglima
- Penggerak Jualan
- Strategi Jenama
- Penggerak Pasaran
- Jurutulis Jualan
- Penulis Cadangan when proposal alignment matters
- Penganalisis if proof or performance framing matters
- Penyemak when collateral could create meaningful commercial risk

## Workflow

1. Capture
- Define audience, sales stage, product or offer, and desired next step
- Use `vault/playbooks/discovery-call.md` first if audience, need, or offer is still unclear
- Use `vault/templates/sales-brief.md` as the working brief

2. Align
- Use Strategi Jenama for message coherence
- Use Penggerak Pasaran for channel and commercial angle
- Pull in Proposal Writing if claims must align tightly with scope language

3. Produce
- Create concise, reusable sales assets
- Emphasize value, proof, and objection handling
- Keep collateral grounded in current capability and offer structure

4. Verify
- Use `vault/checklists/content-copy-verification.md`
- Add `vault/checklists/analytics-verification.md` if proof or performance data is central
- Use Penyemak if the asset carries meaningful promise, positioning, or commercial risk

5. Consolidate
- Update project memory if enablement materials change go-to-market posture
- Write a decision file if a new commercial framing becomes the default standard

## Deliverables

- sales brief
- one-pager or enablement asset
- objection-handling notes
- verification summary and residual commercial risk
