# Launch Flow Playbook

Use this playbook for launches, campaign pushes, landing-page releases, growth experiments, or coordinated public-facing updates.

## Trigger

Use when the output affects traffic, messaging, conversion, onboarding, or external launch visibility.

## Primary Roles

- Panglima
- Strategi Jenama
- Penggerak Pasaran
- Jurutulis Jualan
- Pengkarya Kandungan
- Penjejak Carian
- Senibina Antara Muka
- Tukang Web
- Penganalisis
- Penyemak

## Workflow

1. Frame
- Define audience, offer, goal, and success metric
- Use `vault/templates/campaign-brief.md` when the launch needs a formal brief

2. Align
- Use Strategi Jenama if message architecture or positioning is unclear
- Align offer, proof, CTA, and tone before downstream asset creation

3. Produce
- Copywriting for CTA and page messaging
- Content for supporting assets
- SEO for search intent and metadata
- UI/UX and web roles for page implementation or changes

4. Verify
- Use `vault/checklists/content-copy-verification.md`
- Add `vault/checklists/seo-verification.md` for discoverability work
- Add `vault/checklists/uiux-verification.md` for high-impact flow or page changes
- Use Penyemak if the launch has public-facing risk or conversion sensitivity

5. Measure
- Route to Penganalisis for KPI tracking, funnel review, or experiment readout
- Record what metric should confirm launch success

6. Consolidate
- Update project memory if the launch changes current operating priorities
- Write a decision file if messaging or launch policy changes future work

## Deliverables

- campaign brief
- aligned messaging and assets
- verification summary
- metric owner and success signal
