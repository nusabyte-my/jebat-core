# UI UX Verification Checklist

Use for work led by Senibina Antara Muka or any task affecting layout, interaction flow, navigation, and usability.

## Before Sign-Off

- Primary user task is clear
- Visual hierarchy supports the intended flow
- Responsive behavior was considered
- States are clear: default, loading, empty, error, success
- Accessibility basics were reviewed: labels, focus, contrast, keyboard reachability

## Verification

- The intended flow was checked end to end
- Mobile and desktop were considered where relevant
- Major usability risks are listed
- Penyemak is used if the flow is central to conversion or support load
