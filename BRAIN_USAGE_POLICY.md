# Brain Usage Policy

## Goal

Help Jebat choose when to use entity memory versus daily or long-term memory.

---

## Use entity memory when
- the subject is a named thing
- the subject should be retrievable by name later
- the subject describes a durable component of the stack
- the subject is a person, place, concept, tool, system, workflow family, or skill

Examples:
- Jebat
- Hermes
- OpenClaw Adaptation
- jebat-researcher
- Trust Governance

## Use daily memory when
- the note is session-specific
- it records recent actions, progress, or temporary context
- it may later be promoted but is not yet stable

## Use long-term memory when
- the information is broadly durable
- it affects future behavior or project direction
- it should survive beyond transient logs

## Rule of thumb
- named durable subject -> entity memory
- recent work log -> daily memory
- stable cross-session truth -> MEMORY.md

## Practical note
Entity memory should complement, not replace, daily notes and long-term memory.
It is for structured retrieval, not for recording every tiny event.
