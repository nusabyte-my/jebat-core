# Ultraloop Mode

## Goal

Give Jebat a bounded iterative mode for tasks that benefit from repeated inspect → act → verify cycles.

---

## Use when
- the task benefits from short controlled iterations
- progress can be checked each loop
- the work is reversible or low-risk
- success conditions are clear enough to stop

## Loop pattern
1. inspect current state
2. perform one focused change or action
3. verify result
4. log meaningful outcome if needed
5. stop when success condition is reached or escalation is needed

## Guardrails
- no infinite loops
- no bypassing approvals
- no escalating risk just to keep looping
- stop when evidence is insufficient or user input is needed

## Rule
Ultraloop means disciplined iteration, not uncontrolled autonomy.
